# Core Context Documentation

This directory contains the core context documentation for the project, organized into specialized subdirectories.

## Directory Structure

### [Patterns](patterns/essential-patterns.md)
Core development patterns and guidelines that apply across the project.
- **Key File**: `essential-patterns.md` - Essential patterns for error handling, validation, security, etc.

### [Repo](repo/documentation-standards.md)
Repository management, state schema, and status markers.
- `documentation-standards.md`: Standards for repository documentation.
- `state-schema.md`: Schema for project state tracking.
- `status-markers.md`: Definitions of status markers used in the project.

### [Standards](standards/code.md)
Detailed standards for code, testing, documentation, and analysis.
- `code.md`: Coding standards.
- `tests.md`: Testing standards.
- `docs.md`: Documentation standards.
- `analysis.md`: Analysis framework.
- `patterns.md`: Detailed pattern catalog.

### [System](system/project-overview.md)
System architecture, artifact management, and context guides.
- `project-overview.md`: High-level project overview.
- `artifact-management.md`: How artifacts are managed.
- `context-guide.md`: Guide to the context system.

### [Templates](templates/README.md)
Templates for creating new agents, files, and other artifacts.
- `orchestrator-template.md`: Template for orchestrator agents.
- `subagent-template.md`: Template for subagents.
- `meta-guide.md`: Guide for meta-operations.

### [Workflows](workflows/delegation.md)
Standard workflows for development, review, and task management.
- `delegation.md`: Task delegation workflow.
- `review.md`: Code review workflow.
- `sessions.md`: Session management.
- `task-breakdown.md`: How to break down tasks.
