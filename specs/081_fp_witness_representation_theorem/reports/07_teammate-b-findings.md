# Report 07 (Teammate B): Algebraic/Ultrafilter Approach (Path D) — Mathematical Formalization

## Summary

Path D proposes working in the product Boolean algebra B^D to construct BFMCS bundles
with same-family temporal coherence. This report develops the full mathematical theory,
identifies exactly where the approach works cleanly, and explains where it encounters
fundamental obstacles. The central finding is that the algebraic approach does not
circumvent the core combinatorial difficulty — it merely rephrases it.

## 2. The Product Algebra B^D

### Definition

Let B = LindenbaumAlg and D be the duration type (with LinearOrder, Nonempty, shift closure).

**Definition** (Product algebra). The product Boolean algebra `B^D` is:
```
B^D = { f : D → B }
```
with pointwise Boolean operations:
```
(f ⊔ g)(t) = f(t) ⊔ g(t)
(f ⊓ g)(t) = f(t) ⊓ g(t)
(fᶜ)(t)    = (f(t))ᶜ
⊤(t)       = ⊤
⊥(t)       = ⊥
```

This is a well-known construction: a product of Boolean algebras is a Boolean algebra.

**In Lean 4 terms**, `B^D` is `D → LindenbaumAlg` with the `Pi.instBooleanAlgebra` instance
from Mathlib, or equivalently `∀ t : D, LindenbaumAlg` with pointwise instances.

### Elements as "Truth-Value Functions"

An element `f : B^D` assigns to each time `t : D` a truth-value equivalence class `f(t) : B`.
The notation `⟦φ⟧_t` for the "value of φ at time t" is the constant function
`t ↦ ⟦φ⟧` evaluated at `t`, i.e., the element of B^D obtained by embedding `⟦φ⟧ : B`.

More usefully, given a formula φ and a set of "current formulas" M_t, we want to track
which formulas hold at which times. The natural element associated to φ in B^D is:
```
val(φ) : D → B
val(φ)(t) = ⟦φ⟧  for all t
```
This constant embedding of `⟦φ⟧` into B^D captures the formula's truth value independent
of time position.

## 3. The G-Content Filter in B^D

### Definition

Given an MCS M (viewed as a starting point at time t₀ ∈ D), define the **G-content filter**
`F_G ⊆ B^D` generated by:

For each formula φ with G(φ) ∈ M, add the element:
```
e_{G(φ)} : D → B
e_{G(φ)}(t) = ⊤   if t ≥ t₀
e_{G(φ)}(t) = ⊤   (or anything)  if t < t₀
```
Actually, more precisely: "G(φ) ∈ M means φ holds at ALL future times from t₀."
In B^D, this translates to: for all t ≥ t₀, the component at t should contain ⟦φ⟧.

The filter F_G is generated by the elements:
```
{ f : B^D | ∀ t ≥ t₀, f(t) = ⊤ } ∩ { elements derived from G-formulas in M }
```

More concretely, the G-content filter is:

**Definition** (G-content filter, concrete form). For M an MCS at t₀, define:
```
F_G = the filter generated by { χ_{t,φ} | G(φ) ∈ M, t ≥ t₀ }
```
where `χ_{t,φ} : D → B` is the element that is `⟦φ⟧` at position t and `⊤` elsewhere.

Alternatively (the standard formulation): F_G is the filter generated by all elements
`f ∈ B^D` such that there exists G(φ) ∈ M with f ≤ val(φ) (on all times ≥ t₀).

### This is a Proper Filter

**Claim**: F_G is a proper filter (i.e., ⊥ ∉ F_G).

**Proof sketch**: Any finite intersection from the generators is consistent because:
- A finite set {G(φ₁), ..., G(φ_n)} ⊆ M implies G(φ₁ ∧ ... ∧ φ_n) ∈ M (by G-monotonicity)
- Therefore (φ₁ ∧ ... ∧ φ_n) ∈ M (by temp_t_future: G → id)
- So ⟦φ₁⟧ ⊓ ... ⊓ ⟦φ_n⟧ ≠ ⊥ in B (since M is consistent)
- The pointwise product at each position is therefore ≠ ⊥

The key point: properness follows from M being a consistent MCS combined with
the T-axiom (G(φ) → φ) and closure under finite conjunction.

## 4. F-Obligation Constraints: The Fundamental Mismatch

### F-Obligations Are Existential, Not Universal

For each F(φ) ∈ M, we need:
```
∃ s ≥ t₀, φ ∈ M_s
```
In B^D terms, this is: there exists a time s such that the s-component of the
ultrafilter extension of F_G contains ⟦φ⟧.

This is an **existential** condition on the ultrafilter. Standard filter theory
provides no mechanism to enforce existential constraints — filters capture only
universal (upward-closed, conjunction-closed) conditions.

### Three Candidate Approaches

**Option 1: Skolemization First, Then Filter**

Skolemize F-obligations: for each F(φ) ∈ M, choose a witness time s(φ) ≥ t₀ and
add the element "φ at position s(φ)" to the filter as a definite constraint.

Concretely, add to F_G the element:
```
χ_{s(φ),φ} : D → B  defined by  χ_{s(φ),φ}(t) = ⟦φ⟧ if t = s(φ), else ⊤
```

**Problem**: Skolemization requires CHOOSING the witness times. Different choices for
different formulas may conflict:
- F(φ) ∈ M and F(ψ) ∈ M: choose s(φ) = s(ψ) = s
- But what if G(φ → ¬ψ) ∈ M? Then at time s, we need both φ and ψ, but also ¬ψ. Contradiction.
- Report 06 (Section 4) provides an explicit counterexample: G(F(ψ) → ¬φ) in M means
  we cannot have both φ-resolution and F(ψ)-preservation at the same position.

Skolemization before filter extension does NOT solve the problem — it merely restates
the combined seed inconsistency in algebraic language.

**Option 2: Compactness Argument**

Treat the combined constraint system (G-content + Skolemized F-witnesses) as a
first-order theory and apply compactness.

The constraint system in B^D is:
```
T = {G-content constraints} ∪ {∃ s ≥ t₀ : φ_i at s | F(φ_i) ∈ M}
```

Compactness would say: if every finite subset of T is satisfiable, then T is satisfiable.

**Problem**: Compactness applies to first-order logic, but the BFMCS B^D is not a
first-order structure in the relevant sense. The existential quantifiers range over
the duration type D, and the "satisfiability" we need is ultrafilter membership in B.
Classical Boolean prime ideal theorem does not handle existential constraints.

Furthermore, even if we had a compactness argument, it would not yield SAME-FAMILY
witnesses — the compact model could have witnesses scattered across different components
of B^D without any coherence within a single projection.

**Option 3: Iterative Filter Extension**

1. Start with F_G (G-content filter)
2. Pick one F-obligation F(φ₁) ∈ M
3. Try to extend F_G to include "φ₁ at some s" — find s where this is consistent
4. Add to filter, get F_G' = F_G + "φ₁ at s₁"
5. Repeat for F(φ₂), F(φ₃), ...

This is exactly the chain construction approach (Paths A/B/C in Report 06). It does
not gain anything from being phrased in B^D terms — the same conflicts arise.

**Conclusion**: None of the three approaches resolve the fundamental mismatch between
existential F-obligations and the universal structure of filters.

## 5. Inter-Position Dependencies: The Structural Obstruction

### The g_content Dependency Chain

The BFMCS construction requires that for each family, consecutive MCSs are related by
g_content propagation:
```
g_content(M_t) ⊆ M_{t+1}
```
where `g_content(M) = { φ | G(φ) ∈ M }`.

In B^D terms, this means: if the filter at position t contains ⟦φ⟧ because G(φ) ∈ M_t,
then the filter at position t+1 must ALSO contain ⟦φ⟧.

This is a DEPENDENCY between projections of the ultrafilter. Specifically, if we let:
```
π_t : B^D → B  denote projection to coordinate t
U_t = π_t(U)   denote the ultrafilter at position t
```
then we need: `g_content(U_t) ⊆ U_{t+1}` for all t.

### Why Product Ultrafilters Fail

**Theorem** (Standard result). For a product Boolean algebra B^I and an ultrafilter
U ∈ UF(B^I), the projection `π_t(U) = { a ∈ B | (f ↦ a at t, ⊤ elsewhere) ∈ U }`
is an ultrafilter in B.

However, the projections U_t and U_{t+1} are INDEPENDENT — there is no mechanism
in B^D to enforce `g_content(U_t) ⊆ U_{t+1}`.

**Concrete failure**: Consider the product ultrafilter constructed as follows:
- Take any ultrafilter U₀ in B (corresponding to any MCS M₀)
- Take any ultrafilter U₁ in B (corresponding to any MCS M₁)
- Define U(t) = U₀ for t ≤ 0, U(t) = U₁ for t > 0

This is a perfectly valid element of B^D (as a Nat-indexed product of ultrafilters),
but it satisfies g_content(M₀) ⊆ M₁ ONLY if we chose M₁ specifically to extend
the G-theory of M₀. The product ultrafilter framework provides no guarantee of this.

**Root cause**: The g_content dependency is a DIRECTED GRAPH constraint on the
ultrafilter projections. Standard ultrafilter extension theorems extend a filter to
an ultrafilter in B^D, but they cannot enforce coordination between different coordinates.

### The Algebraic Structure That WOULD Capture Dependencies

The correct algebraic structure is not a product algebra but a **shift algebra** or
**dynamic algebra**. Specifically, we would need:

**Definition** (Required structure). An algebra A over D where:
- Each element f ∈ A assigns truth-values to each (formula, time) pair
- A "shift operation" σ : A → A that advances time by one step
- A "G-projection" that satisfies: G(f) in the filter at t ↔ f is in the filter at all t' ≥ t

This is precisely the structure captured by the STSA (Shift-closed Tense S5 Algebra)
on the Lindenbaum algebra, but it is an algebraic structure on a SINGLE algebra,
not a product algebra over positions.

The shift-closure property of FMCS families (via `shifted_fmcs` in `UltrafilterChain.lean`)
reflects this — a family is the trajectory of a single "state" under iterated shift.
In B^D, this would correspond to an element f where f(t+1) is determined by f(t)
via the STSA shift. But B^D as a product algebra has NO such shift structure —
its elements are arbitrary functions D → B with no temporal relationship enforced.

## 6. Same-Family Guarantee in B^D: Why It Works in Theory but Fails in Practice

### The Appeal of the Single-Algebra Approach

The intuition is: if we construct a SINGLE ultrafilter U in B^D (rather than separate
ultrafilters at each position), then the projections π_t(U) all come from the same U.
This should give same-family coherence automatically, because U is "one object."

This intuition is sound for the MODAL case. In the modal completeness proof, we build
a single ultrafilter U in B (the Lindenbaum algebra), and all modal successors are
other ultrafilters in B related by R_Box. The "same-algebra" property is captured by
R_Box being an equivalence relation (S5).

### Why It Fails for Temporal Coherence

The issue is that temporal forward_F requires:

**Same-family temporal coherence**: F(φ) ∈ U_t → ∃ s > t such that φ ∈ U_s, AND U_t
and U_s are projections of THE SAME family.

In B^D, every ultrafilter U gives projections U_t = π_t(U) at every position. These
projections are all "from the same U," so they are trivially in the same family.

**However**, the question is whether U_s satisfies the temporal theory requirements:
- U_{t+1} must contain g_content(U_t) (the G-persistence requirement)
- U_s must contain φ (the F-resolution requirement)

A generic ultrafilter U in B^D will project to ultrafilters U_t that bear NO
relationship to each other. The g_content dependency is NOT automatically satisfied
by being projections of the same U.

**Precise statement**: Let F be a filter in B^D generated by G-content constraints.
By BPIT, F extends to an ultrafilter U in B^D. The projections π_t(U) are ultrafilters
in B. But there is no reason why g_content(π_t(U)) ⊆ π_{t+1}(U) should hold.

The g_content relationship requires: if G(φ) is in π_t(U), then φ is in π_{t+1}(U).
In B^D, G(φ) ∈ π_t(U) means the element "⟦G(φ)⟧ at position t" is in U.
For φ ∈ π_{t+1}(U), we need "⟦φ⟧ at position t+1" to be in U.

These are DIFFERENT elements of B^D. Being in the same ultrafilter U does not
force their relationship unless U contains the "implication element":
```
"(G(φ) at t) → (φ at t+1)"  in  B^D
```
But this implication element is NOT in the G-content filter (it's a product-level
constraint that does not follow from the pointwise structure).

## 7. What the Algebraic Approach DOES Offer

Despite the fundamental obstacles, the algebraic framing yields useful clarity:

### 7.1 The Boolean Prime Ideal Theorem Suffices for One Step

For a SINGLE temporal step, the BPIT does work:

**Theorem** (F-witness via BPIT, one step). If F(φ) ∈ M (MCS), then there exists
an ultrafilter U in B containing {⟦φ⟧} ∪ {⟦ψ⟧ | G(ψ) ∈ M} ∪ box_theory(M).

This is exactly `temporal_theory_witness_exists` (already proven in `UltrafilterChain.lean`).
The BPIT/Zorn argument produces a single MCS W with the required properties.

The algebraic framing here is: extend the filter generated by
{⟦φ⟧} ∪ {⟦ψ⟧ | G(ψ) ∈ M} to an ultrafilter. This is standard and works because
the generator set is consistent (shown by the G-lift argument).

### 7.2 The Stone Representation Theorem

The correspondence between MCSs and ultrafilters of B (ultrafilter-MCS bijection in
`UltrafilterMCS.lean`) is the Stone representation theorem for the Lindenbaum algebra.
This is fully formalized and provides a clean algebraic perspective on MCSs.

### 7.3 Modal Coherence Is Algebraic

The R_Box equivalence relation on ultrafilters correctly captures S5 modal accessibility.
The algebraic proof that R_Box is an equivalence relation (proven sorry-free via
`R_Box_euclidean`, `R_Box_symm`, `R_Box_trans`) is exactly the Stone-Tarski approach.
For the MODAL case, the algebraic approach works beautifully.

### 7.4 The Gap Is Precisely the Temporal Part

The algebraic approach works for:
- Propositional logic (Boolean algebra, fully formalized)
- Modal S5 logic (Boolean algebra + STSA operators for box, fully formalized)
- Single-step temporal witnesses (BPIT for one F(φ), fully formalized)

It fails for:
- Multi-step temporal coherence with same-family witnesses
- The g_content propagation dependency chain

This clarifies that the "same-family temporal coherence" problem is genuinely a
TEMPORAL property that requires a chain construction, not an algebraic one.

## 9. Why the Algebraic Approach Does Not Overcome the Core Difficulty

### The Reformulation Trap

The algebraic approach reformulates the problem as:
"Extend the G-content filter F_G in B^D to an ultrafilter U with F-obligations satisfied."

But this is equivalent to the original problem:
"Build an FMCS family where every F(φ) at t has a witness φ at some s > t in the SAME family."

The bijection is:
- FMCS family ↔ sequence of ultrafilters in B with g_content coherence
- "Same-family witness" ↔ coordinates of a single ultrafilter in B^D... ONLY if the
  ultrafilter respects g_content coherence, which B^D product construction does not enforce.

### The Product Structure Is Wrong

The fundamental issue is that B^D with the PRODUCT structure treats each time position
as independent. The correct structure for temporal logic is B^D with a SEQUENTIAL
(or SHIFT-CLOSED) structure where position t+1 is constrained by position t via G.

This sequential structure is precisely what the SuccChainFMCS construction provides:
it builds MCSs one step at a time, each seed extending the G-theory of the previous.
The product algebra B^D is the WRONG algebraic framework for this purpose.

### Cost-Benefit Analysis for Lean 4 Implementation

**To implement Path D would require:**
1. Define B^D as a formal structure with temporal dependency constraints (not just Pi.instBooleanAlgebra)
2. Define the "G-coherent filter" in B^D (filter closed under shift: if G(φ) component at t is in, then φ component at t+1 must be in)
3. Prove BPIT for G-coherent filters (NONSTANDARD — not in Mathlib)
4. Prove the G-coherent ultrafilter projects to same-family MCSs
5. Handle F-obligations within the G-coherent filter (same difficulty as before)

Step 3 is a custom ultrafilter extension theorem for filters with sequential dependencies.
This is non-trivial and would require new Lean infrastructure exceeding what Paths A/C require.

**Compared to Path A (F-Persistence Chain):**
- Path A requires: F-content consistency theorem (trivial, Report 06 §7), custom Lindenbaum steps
- Path D requires: G-coherent filter theory, custom BPIT variant, AND the same F-resolution difficulty
- Path D is strictly harder for no gain

## 10. Conclusions

### Path D is Mathematically Sound but Operationally Redundant

The algebraic approach CAN be made to work in principle, but only if we add:

1. **Sequential filter structure**: Replace the product B^D with a filtered colimit or
   sequential structure where each component is constrained by its predecessors via G.

2. **Existential constraint handling**: Either Skolemize F-obligations (which requires
   solving exactly the same scheduling conflicts as Paths A/C) or use a compactness
   argument (which yields a non-constructive ultrafilter that may not project to an
   FMCS family with same-family witnesses).

3. **New Lean infrastructure**: The required algebraic structures are not in Mathlib
   and would require significant new formalization.

### The Correct Path Is Chain-Based

The existing code's identification of `SuccChainFMCS` as the correct vehicle is right.
The chain construction naturally enforces:
- G-content propagation (built into the chain's seed definition)
- Modal saturation (inherited from MCS properties at each step)
- Box-class coherence (via box_theory in the seed)

The only gap is F-obligation resolution within the same chain (same-family forward_F),
which is exactly the blocker in `bfmcs_from_mcs_temporally_coherent`.

### Recommendation

Path D does not offer a simpler route to resolving the sorry. The algebraic language
clarifies the structure of the problem but does not eliminate the combinatorial
difficulty. The effort required to formalize Path D would be substantially greater
than Path A.

**The most promising immediate step** (confirmed by this analysis) is Report 06's
Path A: prove that in the F-persistence chain, F-obligations are eventually resolved.
The key ingredient is showing that if F(φ) persists in all M_n (by F-persistence),
then the g_content grows along the chain until it can no longer derive ¬φ combined
with the surviving F-formulas — at which point φ can be included.

This scheduling argument is not algebraic — it is a temporal induction that requires
the specific structure of the SuccChainFMCS construction.
