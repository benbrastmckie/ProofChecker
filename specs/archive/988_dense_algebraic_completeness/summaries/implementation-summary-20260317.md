## Blockers Identified

### 1. Totality Gap (Critical)

The plan (implementation-002.md) claims:
> "This gives a linear order from the preorder"

But this requires the preorder to be **total**. The current CanonicalMCS preorder:
```lean
le a b := a = b ∨ CanonicalR a.world b.world
```
is NOT total. Two different MCS with no CanonicalR between them are incomparable.

Antisymmetrization of a non-total preorder gives a **PartialOrder**, not LinearOrder.

### 2. Countability Gap (Significant)

The plan claims CanonicalMCS (all MCS) is countable. This is FALSE.

- Set of all MCS has cardinality 2^ℵ₀ (continuum)
- Each MCS is a subset of countable set Formula, but there are uncountably many such subsets
- The TimelineQuot approach works because it restricts to a COUNTABLE reachable subset

## Root Cause Analysis

The plan conflates two different constructions:

| Construction | Has Totality | Has F/P Witnesses | Is Countable |
|--------------|--------------|-------------------|--------------|
| CanonicalReachable (boneyard) | YES | NO (backward_P blocked) | YES |
| CanonicalMCS (all MCS) | NO | YES | NO |

The plan assumes CanonicalMCS has all three properties, but it only has F/P witnesses.

## Possible Resolutions

### Option A: Extend CanonicalReachable with Past

Define a bidirectionally-reachable subset that includes both forward and backward witnesses:
- Start from root M₀
- Include all forward-reachable MCS (via CanonicalR)
- Include all backward-reachable MCS (via CanonicalR_past)
- Prove this subset has totality AND witnesses

Risk: Totality may fail when mixing forward and backward directions.

### Option B: Use Different Ordering

Define a total ordering on CanonicalMCS that is compatible with temporal coherence:
- Perhaps using a well-ordering axiom
- Risk: May not preserve the temporal structure needed for witnesses

### Option C: Revise the Plan

The plan may need fundamental revision. The CanonicalQuot approach may not work as described.

### Option D: Use TimelineQuot with Modal Saturation

Build BFMCS over TimelineQuot where:
- D = TimelineQuot (has linear order, countable, dense)
- Each FMCS family is built from a different root
- Modal saturation provides modal_forward/modal_backward

This is closer to the existing DiscreteInstantiation pattern.
