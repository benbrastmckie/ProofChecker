## 2. What Does "From Syntax" Actually Require?

### 2.1 The Philosophical Distinction

The user draws a line between:

**From syntax (acceptable)**:
- Objects built from formulas, derivations, MCSes, or quotients thereof
- Relations defined by syntactic conditions (e.g., GContent inclusion)
- Constructions that USE the axioms of the proof system to derive properties

**Imported/foreign (unacceptable)**:
- Z, Q, R used as primitive temporal domains
- Countable/uncountable ordinals used as index sets
- Any mathematical structure not derived from the formulas themselves

### 2.2 What Classical Completeness Proofs Actually Do

To ground this analysis, consider how classical completeness proofs handle the model construction:

**Henkin's completeness theorem (first-order logic)**:
- Domain = equivalence classes of TERMS (purely syntactic)
- Relations = defined by formula membership in the canonical theory
- Everything is "from syntax"

**Kripke completeness for modal logics (e.g., Goldblatt 1992)**:
- Worlds = MCSes (sets of formulas -- syntactic)
- Accessibility = GContent inclusion (syntactic condition)
- The model IS the syntax, quotiented and organized

**Tense logic completeness (Burgess 1984, Goldblatt 1992)**:
- Worlds = MCSes (syntactic)
- Temporal order = CanonicalR (syntactic)
- Time points = the MCSes themselves, ordered by CanonicalR
- NO separate "time group D" is introduced -- the time structure IS the ordered set of MCSes

**The JPL paper's approach**:
- Introduces D = Z as a PARAMETER of the frame definition
- The canonical model CHOOSES D = Z (or Q)
- This is a departure from the Goldblatt/Burgess tradition

### 2.3 The Critical Observation

In classical tense logic completeness (Goldblatt, Burgess), there is NO separate duration type D. The frame is (W, R) where R is the temporal accessibility relation, and the completeness theorem proves that the canonical frame (MCSes, CanonicalR) validates exactly the theorems of the logic.

The JPL paper introduces D as part of a RICHER framework (TaskFrame) that adds group structure to the temporal order. This group structure is what forces the question of where D comes from.

**The philosophical tension**: The TaskFrame definition requires D to be an `AddCommGroup` with `LinearOrder` and `IsOrderedAddMonoid`. These are algebraic properties that MCSes (sets of formulas) do not naturally possess. Some bridge between syntax and algebra is unavoidable.

### 2.4 What Counts as "From Syntax" in Practice

A construction is "from syntax" if:

1. **The carrier set** is built from syntactic objects (formulas, MCSes, derivations, or quotients/subsets thereof)
2. **The relations** are defined by syntactic conditions
3. **The algebraic structure** may use universal algebraic constructions (free groups, quotients by congruences) applied to syntactic data
4. **No specific external mathematical object** (like Q, Z, or R) is chosen as the primitive carrier

Under this definition:
- BidirectionalFragment: FROM SYNTAX (carrier = MCSes reachable from a root)
- BidirectionalQuotient: FROM SYNTAX (antisymmetrization of the fragment)
- TranslationGroup = Additive(T ≃o T): FROM SYNTAX (automorphisms of a syntactic structure)
- D = Q: NOT from syntax (Q is imported)
- D = Z: NOT from syntax (Z is imported)

## 5. Alternative Constructions That Preserve Pure-Syntax Spirit

### 5.1 Strategy G: Difference Group of the Quotient

**Idea**: Instead of using the full automorphism group Aut+(T), construct D as the "difference group" of T -- the group generated by formal differences of elements.

**Construction**:
- T = BidirectionalQuotient (from syntax)
- D = Free abelian group on T, quotiented by the relation that encodes the temporal structure
- Specifically: D = the group completion (Grothendieck group) of the ordered monoid (T, +_temporal) where +_temporal is defined by the temporal axioms

**Problem**: T does not naturally have a monoid operation. The temporal order gives us <=, not +. We need to DEFINE addition on T or construct D differently.

**Variant**: D = the free ordered abelian group generated by formal differences [b] - [a] for a <= b in T, subject to:
- [c] - [a] = ([c] - [b]) + ([b] - [a]) whenever a <= b <= c
- [a] - [a] = 0

This is essentially constructing the difference group of a linearly ordered set, which is a well-known construction.

**Assessment**: This IS from syntax (D is generated by differences of quotient elements, which are syntactic). The challenge is that the resulting group may not be the "right" one -- it needs to act on T as a torsor, which requires homogeneity (for any a, b in T, the difference [b] - [a] acts as a translation taking a to b).

**Mathlib support**: Lean/Mathlib does not have a ready-made "difference group of a linearly ordered set" construction. It would need to be built.

### 5.2 Strategy H: T as Torsor Over Its Own Difference Group

**Idea**: Define D abstractly as the type of "syntactic translations" -- formal expressions of the form "from a to b" where a, b are quotient elements.

**Construction**:
1. T = BidirectionalQuotient (from syntax)
2. Define `SyntacticDifference := T x T / ~` where (a,b) ~ (c,d) iff "the translation from a to b equals the translation from c to d"
3. The equivalence ~ is defined by: (a,b) ~ (c,d) iff for all e in T, the unique f with "translation from a to b applied to e gives f" satisfies "translation from c to d applied to e also gives f"

**Problem**: This is circular -- we're defining translations in terms of what translations do. We need an independent characterization of when two pairs represent the same displacement.

**Non-circular version**: (a,b) ~ (c,d) iff there exists an order automorphism sigma of T with sigma(a) = c and sigma(b) = d. This uses the automorphism group, bringing us back to the TranslationGroup approach.

**Assessment**: Strategy H reduces to Strategy G or to the original TranslationGroup approach.

### 5.3 Strategy I: Quotient IS the Time Domain (T = D)

**Idea**: Use T itself as D, with group structure derived from choosing an origin.

**Construction**:
1. T = BidirectionalQuotient (from syntax)
2. Choose origin o = [M0] (the root's equivalence class)
3. Define addition: a + b = the element c such that "the displacement from o to c equals the displacement from o to a plus the displacement from o to b"
4. This requires the action of Aut+(T) to be simply transitive (i.e., T is a torsor over Aut+(T))

**Problem**: This AGAIN requires homogeneity and freeness of the automorphism group action. If these hold, then T is isomorphic to D = Aut+(T) as a torsor, and T inherits a group structure. But these are exactly the deferred items from TranslationGroup.lean.

**Assessment**: Strategy I is viable IF homogeneity can be proven. It IS "from syntax" because T is syntactic and the group operation is derived from the automorphism structure.

4. The TaskFrame connection (if needed) is a SEPARATE theorem showing that any (W, R) frame satisfying certain properties can be presented as a TaskFrame

## 7. Recommended Approach: Strategy K (Syntactic Histories)

### 7.1 Why Strategy K

Strategy K is the best balance of philosophical purity and mathematical feasibility:

1. **It IS "from syntax"**: The carrier is built from specific Lindenbaum witnesses, the ordering is CanonicalR, and Q appears only as a representation target (not a primitive choice).

2. **It avoids the uncountability blocker**: By restricting to specific witnesses (canonical_forward_F, canonical_backward_P), the fragment is countable by construction.

3. **It addresses density constructively**: The DN axiom is used to show that between any two witnesses, an intermediate witness can be constructed. This is mathematically simpler than proving density of the full uncountable quotient because the restricted fragment has more structure.

4. **The group structure transfer is canonical**: Cantor's theorem (isomorphism of countable DLOs without endpoints) provides a unique-up-to-isomorphism identification with Q. The group structure inherited from Q is thus canonical.

5. **Mathlib infrastructure exists**: `Order.iso_of_countable_dense` provides the isomorphism, and Q has all required instances.

### 7.2 Construction Details

**Phase 1: Define the Restricted Fragment**

```
-- Reachability via specific witnesses only
inductive WitnessReachable (M0 : Set Formula) (h0 : SetMaximalConsistent M0) :
    (M : Set Formula) -> SetMaximalConsistent M -> Prop where
  | refl : WitnessReachable M0 h0 M0 h0
  | forward_step {M1 M2} {h1 h2} :
      WitnessReachable M0 h0 M1 h1 ->
      (exists phi, Formula.some_future phi in M1 /\
        M2 = canonical_forward_F_witness M1 h1 phi) ->
      WitnessReachable M0 h0 M2 h2
  | backward_step {M1 M2} {h1 h2} :
      WitnessReachable M0 h0 M1 h1 ->
      (exists phi, Formula.some_past phi in M1 /\
        M2 = canonical_backward_P_witness M1 h1 phi) ->
      WitnessReachable M0 h0 M2 h2
```

This is a SUB-FRAGMENT of BidirectionalFragment. Each step follows a SPECIFIC Lindenbaum witness, not an arbitrary CanonicalR edge.

**Phase 2: Prove Countability**

The restricted fragment has a surjection from `List (Formula + Formula)` (each list element encodes a forward or backward step with a specific formula). Since Formula is countable and List preserves countability, the fragment is countable.

**Phase 3: Prove Totality**

The restricted fragment inherits totality from the full fragment? NO -- we must re-prove it. But the temp_linearity axiom still applies: if M1 and M2 are both forward-reachable from M, they are comparable. The challenge is that the restricted fragment uses SPECIFIC witnesses, so the linearity proof must show that the specific witnesses are comparable.

THIS IS THE KEY DIFFICULTY. The current totality proof (`canonical_forward_reachable_linear`) works for arbitrary CanonicalR edges. For the restricted fragment, we need linearity among the specific Lindenbaum witnesses. Since Lindenbaum witnesses are chosen by `Classical.choice`, their relative ordering is not directly controllable.

However, the linearity argument does not depend on WHICH MCSes M1 and M2 are -- only that CanonicalR M M1 and CanonicalR M M2. Since the specific witnesses satisfy CanonicalR, the linearity theorem applies. So totality DOES transfer to the restricted fragment.

**Phase 4: Prove Density (requires DN)**

Given [a] < [b] in the restricted quotient:
- CanonicalR a.world b.world holds, and NOT CanonicalR b.world a.world
- Extract chi with G(chi) in b.world, chi not in a.world
- F(chi) in a.world (by canonical_F_of_mem_successor)
- Apply DN: FF(chi) in a.world
- So F(chi and F(chi)) in a.world (derivable from FF(chi))
- The specific forward witness c = canonical_forward_F_witness a (chi and F(chi))
  - c is in the restricted fragment (forward step from a)
  - chi and F(chi) in c.world
  - In particular, chi in c.world and F(chi) in c.world
  - chi in c.world implies c != a (in quotient, since chi not in a.world)
  - F(chi) in c.world: now take forward witness d from c for chi
    - chi in d.world, and CanonicalR c.world d.world
  - Need to show c != b (in quotient)

**The density argument for the restricted fragment faces the SAME Lindenbaum collapse issue** as DenseQuotient.lean. The specific forward witness c could equal b (in the quotient) because Lindenbaum might extend {chi and F(chi)} union GContent(a) to exactly b.world. The DN axiom gives us the EXISTENCE of an intermediate MCS but not a SPECIFIC one distinct from the endpoints.

**This means the density blocker persists in the restricted setting.**

## 8. The Density Blocker: Root Cause Analysis

### 8.1 Why Density is Hard

The density property requires: given [a] < [b], find [c] with [a] < [c] < [b].

The difficulty is:
1. We can find formulas distinguishing a and b (chi with G(chi) in b, chi not in a)
2. DN gives FF(chi) in a, so we can construct an intermediate MCS c with F(chi) in c
3. But we cannot ensure [c] is distinct from BOTH [a] and [b] in the quotient
4. Lindenbaum's lemma is existential -- it picks AN extension, and Classical.choice gives no control over which one

### 8.2 The Mathematical Reality

In classical tense logic completeness proofs (Goldblatt, Burgess), density of the canonical model is NOT proven as a separate step. Instead:

- The canonical model for TM (without DN) uses (MCSes, CanonicalR) directly
- Completeness is proven for the class of ALL linear temporal orders
- For TM+DN, the completeness proof shows that the GENERATED submodel (from a single root) is dense, using DN to construct intermediate points

The issue is that "dense" in the relational frame sense (for every a < c, exists b with a < b < c) does not straightforwardly translate to "dense" in the ordered quotient sense when the quotient identifies MCSes with identical G-content.

### 8.3 A Potential Resolution: Goldblatt's Canonical Model for Tense Logic

Goldblatt (1992, Chapter 6) constructs the canonical model for tense logic as:
- W_c = set of all MCSes (not a fragment from a single root)
- R_c = CanonicalR (GContent inclusion)
- The canonical model has a CONNECTED component for each MCS

For TM+DN completeness, Goldblatt shows:
- If phi is NOT a theorem, there exists an MCS Gamma containing not-phi
- The CONNECTED COMPONENT of Gamma under R_c forms the canonical model
- This component, ordered by R_c, is a DENSE linear order (using DN)

The density argument works because Goldblatt considers the FULL connected component (all MCSes reachable by arbitrary CanonicalR chains), not a restricted fragment. The density uses the fact that between any two CanonicalR-related MCSes with distinct G-content, DN forces the existence of an intermediate MCS with STRICTLY intermediate G-content.

**The key difference**: Goldblatt does NOT quotient by G-content equivalence. He works with MCSes directly, and the "temporal order" is CanonicalR itself (which is a linear preorder on the connected component). Density is proven at the MCS level (which is easier because no quotient collapse can occur -- every new MCS is a new point).

### 8.4 The Resolution: Work with the Preorder, Not the Quotient

**Insight**: The density blocker arises from working with the QUOTIENT (BidirectionalQuotient = Antisymmetrization). If instead we work with the PREORDER (BidirectionalFragment with its Preorder instance), density is provable:

Given a < b in the fragment (CanonicalR a.world b.world, NOT CanonicalR b.world a.world):
- Extract chi with G(chi) in b.world, chi not in a.world
- F(chi) in a.world (canonical_F_of_mem_successor)
- Apply DN: FF(chi) in a.world, giving F(chi and F(chi)) in a.world
- Construct intermediate c via canonical_forward_F with formula (chi and F(chi))
- c.world contains chi and F(chi)
- CanonicalR a.world c.world (by construction)
- From F(chi) in c.world, construct d via canonical_forward_F with formula chi
- CanonicalR c.world d.world, chi in d.world
- Need: a < c. We have CanonicalR a.world c.world. Need NOT CanonicalR c.world a.world.
  - chi in c.world but chi not in a.world, and chi in GContent(c) would give chi in a if CanonicalR c a. So if chi in c.world and G(chi) in c.world, then CanonicalR c a would give chi in a. BUT we only know chi in c.world, not G(chi) in c.world.
  - HOWEVER: c was constructed as Lindenbaum({chi and F(chi)} union GContent(a)). So GContent(a) subset of c.world. This gives CanonicalR a.world c.world. For the reverse: we need to show GContent(c.world) is NOT a subset of a.world.

**This is STILL the same difficulty**. The issue is showing that c.world has G-content strictly between a and b. The Lindenbaum extension can add arbitrary G-formulas to c.world beyond what we control.

## 9. A Fresh Approach: D as Formal Displacements

### 9.1 The Core Insight

Rather than trying to extract D from the ORDER STRUCTURE of MCSes (which hits the density/automorphism blockers), we can construct D from the PROOF THEORY of displacements.

Define:
- A **formal displacement** is a finite sequence of F-steps and P-steps (i.e., a word in the free group on {F, P})
- Two formal displacements are equivalent if they produce the same temporal displacement in ALL models of TM (or TM+DN)

**D = Free group on {F, P} / semantic equivalence**

This is purely syntactic:
- The carrier is built from the syntax (sequences of F/P operators)
- The equivalence is defined by the axioms of TM
- The group operation is concatenation of displacement sequences
- The identity is the empty sequence
- The inverse of an F-step is a P-step

### 9.2 Why This Works

The axioms of TM determine the equivalence relation:
- temp_4: G(phi) -> GG(phi) implies FF ~ FF (two forward steps are non-trivially related)
- temp_A: F(phi) -> GF(phi) implies the ordering on displacements
- temp_K: G distributes over conjunction
- temp_linearity: ensures the displacement group is ORDERED

For TM alone: D is a totally ordered group (possibly non-abelian, but Holder's theorem from temp_linearity forces abelianness)

For TM+DN: DN (F -> FF) implies that between any two displacements, there is an intermediate one. This makes D dense.

For TM+DP/DF: DP/DF force D to be discrete.

### 9.3 Formalization Challenges

This approach is conceptually clean but hard to formalize:
1. "Semantic equivalence" of displacements requires quantifying over all models -- this is a meta-theoretic definition
2. The resulting quotient group needs to be shown to have a linear order
3. The connection between D and the canonical model needs to be established

**Estimated effort**: Very high (30+ hours). This is essentially building a universal algebra / Lindenbaum-Tarski algebra for the temporal fragment.

### 9.4 Assessment

Strategy I (formal displacements) is the MOST philosophically pure approach but also the most technically demanding. It requires building substantial new infrastructure (term algebras, semantic equivalence, quotient group construction).

   - Q is not imported -- it is DISCOVERED as a property of the syntactic construction
   - The group structure on D is INDUCED by the isomorphism, not assumed

### 10.3 Why This Satisfies the User's Constraints

- D is "from syntax" because it IS the set of MCSes (or their quotient)

  - TM + DN: the connected component IS dense (proven using DN), so isomorphic to Q

## 11. Summary of Findings

1. **The philosophical constraint is already met** by the TranslationGroup construction -- BidirectionalQuotient and its automorphisms are "from syntax." The blockers are mathematical, not philosophical.

2. **The layered extension approach is conceptually sound** but faces the same mathematical blockers (density, countability, Holder's theorem) at each layer.

3. **The density blocker is the root issue** and persists across all strategies that use BidirectionalQuotient. It arises from Lindenbaum collapse: the inability to control WHICH MCS the Lindenbaum extension produces.

4. **Working at the MCS/preorder level (not the quotient level) may avoid the collapse issue**, since every new MCS is a genuinely new point (no equivalence class collapse).

5. **The recommended approach (K-Relational)** proves completeness for relational frames first (standard, from syntax), then uses Cantor's theorem to DISCOVER that the canonical model is isomorphic to Q. This preserves the "from syntax" spirit while using Q only as a characterization.

6. **Zero-sorry feasibility**: The K-Relational approach has a reasonable chance of zero sorries IF density at the MCS level can be proven. This is the critical open question that should be investigated next.
