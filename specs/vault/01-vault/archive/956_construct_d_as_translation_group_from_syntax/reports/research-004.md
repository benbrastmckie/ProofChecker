## 1. Executive Summary

This report provides a deep analysis of the proof strategy for Step 5 of the translation group construction: proving that the evaluation-at-origin map `eval_0 : D -> T` (where `eval_0(d) = d(w_0)`) is a bijection. This step requires proving **transitivity** (surjectivity) and **freeness** (injectivity) of the action of D = Aut+(T) on T.

**Critical finding**: The naive approach (D = all order automorphisms of T) has a fundamental gap. **Freeness fails for general linear orders** -- for example, `x -> 2x` is an order automorphism of Q fixing 0 but not equal to the identity. The correct approach requires either:

1. **Restricting D** to a specific subgroup of Aut+(T) for which freeness holds, or
2. **Proving T has special structure** (e.g., T is order-isomorphic to Z) that forces Aut+(T) to consist only of translations, or
3. **Redefining D** directly as a torsor/group on T itself using the origin, bypassing the automorphism group entirely.

**Recommended strategy**: Option 3 (direct torsor construction). Define D = T as a set, with group structure induced by the origin w_0. This completely sidesteps the homogeneity/rigidity problems and yields a clean formalization using Mathlib's `AddTorsor` infrastructure.

## 2. The Problem: Why Naive Aut+(T) Fails

### 2.1 Freeness Gap

Research-001 claims freeness: "if d(w_0) = w_0 then d = id." The informal argument is: "d preserves order and fixes w_0, so it fixes every reachable point."

**This argument is incorrect for general linear orders.** Here is a concrete counterexample:

- Let T = Q (rationals) with the standard order.
- Let d(x) = 2x. This is an order-preserving bijection.
- d(0) = 0, but d is not the identity.
- The action of Aut+(Q) on Q is NOT free.

For the action to be free, we need a structural property of T that Q lacks. The key candidate properties are:

| Property | Definition | Example | Aut+(T) free? |
|----------|-----------|---------|---------------|
| Discrete | Every element has immediate successor/predecessor | Z | Yes |
| Dense, no endpoints | Between any two elements there's another | Q | No |
| Dense + complete | Dense + every bounded set has sup | R | No (x -> 2x) |
| Discrete + Archimedean | Discrete and generated by a single step | Z | Yes |

**Conclusion**: Freeness requires T to be **discrete** (or to restrict to a specific subgroup of translations).

### 2.2 Transitivity Gap

Research-001 claims transitivity: "for any w in T, there exists d in D with d(w_0) = w." The informal argument is: "T is homogeneous -- no point is structurally distinguishable."

**This is plausible but non-trivial to prove.** Homogeneity of T means that for any two points a, b in T, there exists an order automorphism mapping a to b. This is known to hold for:

- Z (translations x -> x + n)
- Q (Cantor's theorem: any two countable dense linear orders without endpoints are isomorphic, and the isomorphism can be chosen to map any point to any other)
- R (does NOT hold: Aut+(R) = {x -> ax + b : a > 0}, which is transitive, but this is non-trivial)

For the MCS timeline T: homogeneity would follow if we can show T is isomorphic to Z (discrete case) or Q (dense case). Without such an isomorphism, homogeneity requires a direct construction of automorphisms, which is challenging.

### 2.3 The Fundamental Issue

The approaches in Sections 2.1-2.2 share a common problem: **they try to characterize Aut+(T) abstractly, then derive properties of its action.** This is backwards. The right approach is:

**We know the structure of T concretely** (it is built from MCSes via the existence lemma). We should use this concrete structure to **directly construct** the group operation on T, rather than going through the automorphism group.

## 3. The Correct Approach: Direct Torsor Construction

### 3.1 Mathematical Framework

A **torsor** (or principal homogeneous space) for a group G over a set P is a free transitive G-action on P. Equivalently, it is a structure where:
- For any two points p, q in P, there is a unique group element g with g.p = q
- This g is written q - p (the "displacement" from p to q)

The key insight: **any pointed torsor is canonically a group**. Given (P, -, p_0), define:
- Identity: p_0
- Addition: p + q = the unique r such that r - p_0 = (p - p_0) + (q - p_0)
- In practice: p + q is the point whose displacement from p_0 equals the sum of displacements

For the MCS timeline T with origin w_0:
- We DON'T need to define D = Aut+(T) first
- We CAN define the group operation directly on T
- The group operation encodes "temporal displacement composition"

### 3.2 Concrete Construction on T

**Step A: Define the displacement operation.**

For w, v in T, define `v - w` to be the unique order automorphism d of T such that d(w) = v. But this requires proving existence and uniqueness, which is exactly the transitivity/freeness problem we're trying to avoid!

**Step B: Alternative -- use the concrete MCS structure.**

For the MCS timeline, the "successor" operation is given by the existence lemma: given w and an F-formula, we get a witness w' > w. The "shift" from w_0 to w can be defined as:

```
shift_w : T -> T
shift_w(v) = "the point at the same position relative to w as v is relative to w_0"
```

But formalizing "same position relative to" again requires the group structure we're trying to build.

### 3.3 The Breakthrough: T is Order-Isomorphic to Z

The correct path is to **prove T is order-isomorphic to Z**, then use Z's group structure.

**Why T should be isomorphic to Z:**

1. **T has no endpoints**: For any w in T, the existence lemma gives w' > w (from seriality: G(phi) -> F(phi) applied to G(w_0 member formulas)) and w'' < w (similarly from H(phi) -> P(phi)).

2. **T is discrete**: In T (the antisymmetrized quotient), each equivalence class [w] has an **immediate successor** -- the class of the F-witness for a suitable formula. The key argument:
   - The existence lemma for F(phi) gives a one-step successor.
   - Linearity ensures no point lies strictly between [w] and this successor.
   - This is because the MCS construction builds T by **individual witness steps**, each producing an adjacent element.

3. **T is countable**: Each element is reached by finitely many witness steps from w_0, and there are countably many formulas.

4. **Any countable discrete linear order without endpoints is isomorphic to Z**: This is a standard order-theory result.

**However**, proving discreteness of the quotient T is non-trivial. The problem: after antisymmetrization, the quotient classes might collapse multiple "steps" into one, potentially creating gaps or density.

### 3.4 Revised Strategy: Bypass the Z-Isomorphism

Given the difficulty of proving T = Z directly, here is a more practical approach.

**Strategy: Define D = T as a set, with group operation defined using the origin w_0.**

This is valid because:

1. T is a linear order (proven: `instLinearOrderBidirectionalQuotient`).
2. T has a designated point w_0 (the origin).
3. We need `AddCommGroup D`, `LinearOrder D`, and `IsOrderedAddMonoid D`.
4. `LinearOrder D` is already available (since D = T as a type).

The challenge reduces to: **define an `AddCommGroup` structure on T using w_0**, and prove it is ordered.

**Key observation**: We don't actually need eval_0 to be a bijection between an external D and T. We can **define D to be T** and have eval_0 be the identity function! The group structure on D = T is what we need to construct.

## 4. Proof Strategy: Constructing the Group on T

### 4.1 The Abstract Framework

Given a linearly ordered set (T, <=) with a designated point `origin : T`, we want to define an abelian group structure on T such that:
- `origin` is the identity (zero element)
- The order is translation-invariant: a <= b implies a + c <= b + c

**Mathematical requirement**: We need to define `+`, `0`, and `-` on T satisfying the group axioms. The natural candidate:

```
0 = origin
a + b = "the point c such that the displacement origin -> c equals
         the sum of displacements origin -> a and origin -> b"
```

But defining this directly requires a notion of "displacement" that we're trying to construct!

### 4.2 Breaking the Circularity

The circularity can be broken if we have a **successor function** `succ : T -> T` that is an order automorphism.

Given `succ`, define:
- `n : Z -> T` by `n(0) = origin`, `n(k+1) = succ(n(k))`, `n(k-1) = succ^{-1}(n(k))`
- This gives an order isomorphism `Z -> T` (if succ has no fixed points and generates all of T)
- Transfer the group structure from Z to T

**So the problem reduces to: construct a canonical `succ : T -> T`.**

### 4.3 Constructing the Successor Function

For the MCS timeline, a natural successor candidate exists. Given [w] in T (an equivalence class of MCSes):

**Attempt 1**: Pick any F-formula `F(phi) in w` and take the witness MCS from `canonical_forward_F`. Problem: the choice depends on phi, and different phi might give different witnesses (or the same equivalence class).

**Attempt 2**: Define succ([w]) = the LEAST element of T strictly greater than [w]. If T is discrete, this exists. But proving T is discrete is the problem we're trying to solve.

**Attempt 3**: Use a SPECIFIC formula to define the successor. For example, the formula `F(top)` (there exists a future moment). Since `top in w` for every MCS w (by `theorem_in_mcs`), and `G(top) in w` (since `top` is a theorem so `G(top)` is derivable by necessitation), we have `F(top) in w` (from `G(phi) -> F(phi)` seriality, or just because `top in w` and `G(top) -> F(top)`).

Wait -- `F(top)` is `neg(G(neg(top)))` = `neg(G(bot))`. Since `G(bot)` implies `bot` (by temp_t_future), and `bot not in w`, we have `G(bot) not in w`, so `neg(G(bot)) in w`, so `F(top) in w`.

The witness for `F(top)` gives us a SPECIFIC MCS `w'` with `CanonicalR w w'` and `top in w'` (which is trivially true). But the witness is NOT unique -- Lindenbaum's lemma uses Choice and the result depends on the enumeration.

**This is a fundamental issue**: the canonical model construction does not produce a deterministic successor function. Each application of the existence lemma involves a non-deterministic Choice step.

### 4.4 The Way Out: Use All Witnesses, Not a Single Successor

The correct approach recognizes that we DON'T need a single successor function. Instead:

**Strategy**: The BidirectionalFragment already gives us a total preorder. After antisymmetrization, T is a linear order. We DON'T need to prove T is isomorphic to Z. Instead, we use the following:

**Theorem (Key Claim)**: For any linear order L without endpoints, the map
```
eval_w0 : Aut+(L) -> L,  f |-> f(w_0)
```
is a bijection if and only if L is a **torsor** for its automorphism group.

**Alternative formulation**: eval_w0 is a bijection iff:
1. The action is transitive (for every w, there exists f with f(w_0) = w)
2. The action is free (f(w_0) = w_0 implies f = id)

As we showed in Section 2, freeness fails for general linear orders. So we MUST restrict D.

### 6.2 Surjectivity (Transitivity) Proof Strategy

**Claim**: For every `t : T`, there exists `d : T ≃o T` with `d w_0 = t`.

**Proof approach**: Construct the order isomorphism explicitly.

**Key idea**: Given w_0 and t in T, we need an order-preserving bijection d : T -> T sending w_0 to t. We construct d by "shifting" T.

For the MCS timeline, the shift should map each MCS w to "the MCS at the same position relative to t as w is relative to w_0."

**Formal construction** (assuming T is a discrete linear order without endpoints):

1. Define the "distance" function: for a, b in T with a <= b, let dist(a, b) = the number of immediate successor steps from a to b. (This is well-defined if T is discrete.)

2. Define shift_n : T -> T for each integer n by: shift_n(a) = the point n steps from a. (n positive = forward, n negative = backward.)

3. Let n_0 = dist(w_0, t) (signed distance from w_0 to t).

4. Define d = shift_{n_0}. Then d(w_0) = t.

**But this requires proving T is discrete!** Specifically, we need:
- Every element of T has an immediate successor (element s with a < s and no x with a < x < s).
- Every element has an immediate predecessor.

**How to prove T is discrete**: This is the crux.

**Argument**: In the BidirectionalFragment (before quotient), each element w has a "canonical forward witness" -- the MCS w' given by `canonical_forward_F w w.is_mcs top (...)`. In the quotient, [w'] is either equal to or strictly greater than [w]. The question is: can there be a quotient class strictly between [w] and [w']?

**Claim**: No. Here is why.

Consider [w] and [w'] where w' is the F(top)-witness at w (so CanonicalR w w', i.e., GContent(w) subset w'). Suppose [u] is a quotient class with [w] < [u] < [w']. Then:
- CanonicalR w u (since [w] <= [u])
- CanonicalR u w' (since [u] <= [w'])
- NOT (CanonicalR u w) (otherwise [u] = [w])
- NOT (CanonicalR w' u) (otherwise [u] = [w'])

Now, u is an MCS in the bidirectional fragment. It was produced by some sequence of witness steps from w_0.

**Problem**: We cannot rule out the existence of such u in general! The bidirectional fragment contains ALL MCSes reachable from w_0 by ANY sequence of F/P witness steps. There might be an MCS u that is temporally between w and w' but was reached by a completely different path.

**This means discreteness of T is NOT obvious and may be FALSE.**

### 6.3 Density vs Discreteness of T

**Surprising conclusion**: The MCS timeline T might actually be **dense** (between any two distinct points, there is another). Here is why:

- The axiom system includes NO density axiom (no explicit "between any two moments there is another").
- BUT the axiom system also includes NO discreteness axiom (no "every moment has an immediate successor").
- The BidirectionalFragment is constructed by taking ALL MCSes reachable from w_0 by iterated F/P witness steps.
- Each F-formula `F(phi)` at w gives a DIFFERENT witness (in general), so the fragment can be very rich.
- Between w and the F(top)-witness w', there might be an MCS u reached via a different formula.

**Example**: Suppose `F(p)` and `F(q)` are both in w, where p and q are distinct atomic propositions. The linearity axiom gives three cases:
1. F(p and q) in w -- witnesses coincide
2. F(p and F(q)) in w -- p-witness comes before q-witness
3. F(F(p) and q) in w -- q-witness comes before p-witness

In cases 2 and 3, there are TWO distinct witness MCSes strictly between w and some point, and they may be comparable with each other.

**This suggests T could be dense** (at least, discreteness is not guaranteed by the axioms). If T is dense, then it is order-isomorphic to Q (countable dense linear order without endpoints, by Cantor's theorem).

### 6.4 Implications for the Construction

If T is dense (like Q):
- Aut+(T) is NOT free (x -> 2x fixes the origin)
- D cannot be all of Aut+(T)
- We need a different approach

If T is discrete (like Z):
- Aut+(T) consists only of "shift by n" maps
- Aut+(T) is isomorphic to Z
- The action is free and transitive
- eval_0 is a bijection

**The density/discreteness question is a genuine mathematical issue** that the informal arguments in research-001 gloss over.

## 8. Detailed Proof Sketch for Strategy 1 (Discreteness)

### 8.1 The Discreteness Lemma

**Lemma (discrete_successor)**: For every [w] in T, the quotient class of the F(top)-witness at w is the immediate successor of [w].

**Proof sketch**:

Let w be an MCS in the bidirectional fragment, and let w' = canonical_forward_F(w, top, h_F_top). So:
- GContent(w) subset w' (i.e., CanonicalR w w')
- top in w' (trivially)
- w' is an MCS constructed by Lindenbaum's lemma extending {top} union GContent(w)

We need: there is no MCS u in the fragment with [w] < [u] < [w'].

**Attempt**: Suppose u is such an MCS. Then:
- CanonicalR w u (from [w] <= [u])
- CanonicalR u w' (from [u] <= [w'])
- NOT CanonicalR u w (otherwise [u] = [w])

From CanonicalR w u: GContent(w) subset u.
From NOT CanonicalR u w: GContent(u) is NOT a subset of w.
So there exists phi with G(phi) in u but phi not in w.

From CanonicalR u w': GContent(u) subset w'.
So G(phi) in u implies phi in w'.

Now the question: can such a u exist? The existence of phi with G(phi) in u, phi not in w, but everything in GContent(w) also in u, is a question about the relative strength of the temporal theories.

**Problem**: This argument does NOT lead to a contradiction! In fact, u could have STRICTLY MORE G-formulas than w (i.e., GContent(w) is a strict subset of GContent(u)), which would make [w] < [u] genuinely strict.

**This means the F(top)-witness is NOT necessarily an immediate successor.** There can be intermediate quotient classes.

### 8.2 Reassessing Discreteness

The analysis above shows that **T might not be discrete**. The F(top)-witness at w is one step forward in the pre-quotient, but in the quotient, there might be other classes between [w] and [w'].

**However**, there is a subtlety: the F(top)-witness w' is constructed to extend {top} union GContent(w) to an MCS. The resulting w' has GContent(w) subset w' but its own GContent might be different. A hypothetical intermediate u would need:

- GContent(w) subset u (so u knows everything w says about the future)
- GContent(u) not subset w (so u has additional future knowledge)
- GContent(u) subset w' (so w' is still ahead)

For u to exist in the bidirectional fragment, it must be reachable from w_0. It could have been reached by a completely different path through the MCS space.

**Conclusion**: Discreteness of T is a genuine open question in this axiom system. It likely depends on the specific axioms (density or lack thereof).

### 8.3 What Happens if T is Dense?

If T is dense (isomorphic to Q):
- Aut+(Q) is much larger than Z
- Aut+(Q) acts transitively but NOT freely
- eval_0 is NOT a bijection for D = Aut+(T)
- We MUST use a subgroup of Aut+(T) -- specifically, the group of translations (x -> x + c for some c in Q)
- The group of order-preserving bijections of Q that commute with ALL other such bijections consists exactly of the translations
- This subgroup IS isomorphic to (Q, +)
- This subgroup DOES act freely and transitively

**Key theorem**: For any dense linear order without endpoints L isomorphic to Q, the subgroup of Aut+(L) consisting of translations (maps that commute with all order automorphisms) is isomorphic to (Q, +) and acts freely and transitively.

**But**: Proving this for our specific T requires first proving T is isomorphic to Q, which is as hard as proving T is isomorphic to Z.

### 9.3 What's Missing from Mathlib

1. **No theorem**: "order automorphism of Z fixing 0 is identity" (specific to Z's structure).
2. **No theorem**: "countable discrete linear order without endpoints is isomorphic to Z".
3. **No theorem**: "group of order-preserving translations of a linear order is abelian".
4. **No AddTorsor instance for order automorphisms on linear orders** (because it requires freeness).

## 11. Conclusions and Recommendations

### 11.1 The eval_0 Bijection is Genuinely Hard

The proof that eval_0 is a bijection is not a routine formalization task. It requires resolving the discreteness/density question for T, which is a genuine mathematical question about the axiom system. The informal arguments in research-001 have real gaps.

### 11.2 The Core Difficulty Map

```
eval_0 bijective
├── Surjectivity (transitivity of action)
│   └── Homogeneity of T
│       └── Either: T ≅ Z (discrete case)
│           Or:     T ≅ Q (dense case, need translation subgroup)
│           Either way: need classification of T's order type
│
└── Injectivity (freeness of action)
    └── Rigidity: fixing w_0 forces identity
        └── For D = Aut+(T): FAILS if T is dense (Q case)
            For D = Translations(T): holds but need to define "translation"
            For D = Aut+(Z): holds (Z has no non-trivial order autos fixing 0)
```

### 11.3 Recommended Next Steps

1. **Investigate discreteness**: Determine whether T (the BidirectionalQuotient) is discrete or dense. This is the key mathematical question. If discrete, the Z-isomorphism path is viable. If dense, a fundamentally different approach is needed.

2. **If discrete**: Prove the immediate successor lemma (Section 8.1) and use it to construct the Z-isomorphism.

3. **If undetermined/dense**: Consider the pragmatic approach (Strategy 2 in Section 7.2) -- keep D = Int and focus on fixing the F-persistence sorry directly at the BidirectionalFragment level.

4. **For implementation planning**: The eval_0 bijection should NOT be on the critical path for completeness. It is an elegant theoretical construction but is not required for the core sorry elimination. The critical path is: fix F-persistence at the fragment level -> sorry-free BFMCS construction -> sorry-free completeness.

### 11.4 Summary of Findings
