# Processes Context README

## Purpose

Language-agnostic workflow templates for research, planning, and implementation phases. These define the standard execution patterns used by agents regardless of target language (Lean, LaTeX, general).

## Files

| File | Content |
|------|---------|
| `research-workflow.md` | Research execution: language routing, tool selection, report creation |
| `planning-workflow.md` | Plan generation: phase decomposition, risk analysis, verification criteria |
| `implementation-workflow.md` | Implementation execution: phase checkpoints, resume patterns, summary creation |

## When to Load

Load these files when:
- Understanding how commands execute internally
- Debugging workflow issues
- Extending workflows with new phases or features
- Creating new language-specific workflow variants

## Related Context

- `project/meta/` - System-building guidance for creating new workflows
- `project/lean4/processes/` - Lean-specific workflow extensions
- `project/logic/processes/` - Logic-specific proof construction workflows
- `core/standards/` - Standards that workflows must follow
